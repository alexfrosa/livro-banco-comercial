Ao longo deste livro, você encontrou simulações em cada parte temática. Este capítulo explica como tirar o máximo proveito dessas simulações e como elas se conectam ao conteúdo teórico.

## O que é o simulador

O simulador é um ambiente interativo que replica, de forma simplificada, os sistemas e processos de um banco real. Ele permite que você:

- **Experimente** os conceitos sem risco real
- **Cometa erros** e aprenda com eles
- **Explore** cenários que raramente acontecem na prática
- **Desenvolva intuição** sobre como os sistemas se comportam

O simulador não é um sistema bancário real — é uma abstração educacional. Alguns detalhes foram simplificados para facilitar o aprendizado. Mas os princípios fundamentais são os mesmos.

## Como as simulações se conectam ao conteúdo

Cada simulação foi projetada para reforçar os conceitos do capítulo correspondente:

| Simulação | Conceitos reforçados |
|---|---|
| KYC Onboarding | Validação de dados, análise de risco, decisão de aprovação |
| Ciclo de vida da conta | Estados da conta, bloqueios, encerramento |
| Lançamentos contábeis | Partidas dobradas, imutabilidade, reconciliação |
| Fluxo de pagamentos | Validação, reserva, liquidação, tratamento de falhas |
| Crédito | SAC vs Price, inadimplência, renegociação |
| Tarifas | Regras de elegibilidade, franquias, contestações |
| Liquidez | Gestão intraday, operações de ajuste |
| ALM | Descasamento, duration, impacto de juros |
| AML | Detecção de padrões, investigação, reporte |
| Contabilidade | Fechamento, reconciliação, correção de divergências |
| Investimentos | Aplicação, custódia, resgate, IR |
| Cenários críticos | Diagnóstico e resolução de problemas complexos |

## Dicas para usar as simulações

**Leia o capítulo antes de simular:** as simulações fazem muito mais sentido quando você já entende os conceitos. Não pule direto para a simulação.

**Explore os cenários de erro:** os cenários de falha e erro são tão importantes quanto os cenários de sucesso. É nos erros que você aprende como os sistemas realmente funcionam.

**Observe os logs e registros:** cada simulação mostra os registros gerados (lançamentos contábeis, eventos, logs de sistema). Preste atenção neles — eles mostram o que está acontecendo "por baixo dos panos".

**Tente quebrar o sistema:** tente fazer operações inválidas, inserir valores extremos, simular falhas. Ver como o sistema responde a situações anômalas é muito educativo.

**Conecte com a teoria:** quando algo acontece na simulação, pergunte-se: "por que isso acontece? Qual é o princípio por trás disso?"

## O simulador completo

Além das simulações individuais de cada capítulo, o livro inclui um **simulador completo** que integra todos os módulos. Nele, você pode:

- Criar um banco do zero
- Cadastrar clientes e abrir contas
- Processar pagamentos e ver os lançamentos contábeis
- Gerenciar crédito e investimentos
- Monitorar a liquidez e o ALM
- Fechar o dia contábil

O simulador completo é a melhor forma de entender como todas as peças se encaixam — como uma operação de Pix afeta a liquidez, que por sua vez afeta o ALM, que por sua vez afeta as decisões de captação.
